Institutional Workflows

Built for How Institutions
Actually Work

Three end-to-end workflows designed for real use cases — not demo scenarios.

01 — INVESTMENT
Underwriting
PERSONA: INVESTOR / PE FUND
1
Analytics → filter sub-market (HITEC City, Office A+), review vacancy/absorption/rent trends
2
Property Records → filter, add 3–5 buildings to Compare Tray → side-by-side canvas
3
Building Profile → Lease Roll, Stacking Plan, Owner tab, Loan maturity, Change Log
4
Sales Comps → last 24 months in sub-market → establish ₹/sqft and cap rate benchmarks
5
Create Report → "Investment Underwriting Pack" → auto-populates → export PDF or XLSX
02 — LEASING
Broker Prospecting
PERSONA: BROKER / AGENT
1
Tenants → Lease Expirations → filter: Bengaluru, IT/GCC, 12–18 months, 50k+ sqft
2
Filter by Growth Signals: hiring surge, headcount +12% YoY, new GCC mandate flags
3
Listings → Whitefield, 80k–150k sqft contiguous, Grade A, available <6 months → Compare
4
Owner Tab: identify entity, contact, portfolio; flag owners with loan maturity <12 months
5
Export Outreach List CSV: [Building, Owner, Contact, Sqft, Asking Rent, Loan Maturity]
03 — DISTRESS
Distress Pipeline
PERSONA: INVESTOR / ANALYST
1
Distress Hub → filter: Mumbai, SARFAESI, Office/Industrial → sort auction_date ascending
2
Review reserve price vs. auto-computed market value delta from comps (discount flag)
3
Case Record → property linkage, lender, borrower, DRT/IBC refs, evidence pack
4
Cross-link to Property Record → encumbrances, prior sales, building condition
5
Watchlist → set alerts: reserve price drop, new doc, auction date change, status change

API Reference

22+ REST Endpoints.
Search → Export.

Unified search across all entity types. Property, building, tenant, owner, loan, distress — all cross-linked. Compare tray and scheduled report export included.

Endpoint Method Description
/api/v1/searchGETUnified typeahead search across properties, tenants, owners, loans, distress
/api/v1/propertiesGETList/filter canonical property records with confidence filter
/api/v1/properties/:idGETProperty detail — buildings, transactions, owners, loans, changelog
/api/v1/buildings/:idGETBuilding profile — units, comps, stacking plan, evidence, changelog
/api/v1/listingsPOSTCreate listing with inline evidence pack submission
/api/v1/listings/:idPATCHUpdate listing fields; triggers confidence recalculation
/api/v1/transactions/salesGETSales comps with filters: city, sub-market, date, cap rate, area, verified
/api/v1/transactions/leasesGETLease activity — includes expiration heatmap data endpoint
/api/v1/tenantsGETTenant directory with sector, city, sqft, expiry window, growth signal filters
/api/v1/ownersGETOwner directory with portfolio sqft, AUM, loan maturity, distress flags
/api/v1/loansGETLoan register — maturity schedule, LTV, status (Current/Watch/NPA)
/api/v1/distressGETDistress feed filtered by route (SARFAESI/IBC/DRT/SEBI), city, asset
/api/v1/evidencePOSTSubmit field-level evidence item for any entity
/api/v1/comparePOSTCompare tray — resolve N building/property IDs into comparison grid
/api/v1/reports/:id/exportGETExport saved report as PDF (board-ready) or XLSX (model-ready)
/api/v1/alertsPOSTAlert subscriptions — triggers: confidence drop, expiry, distress status, new listing
